## Step 6: Create or Auto-Generate an Assignment

**You Have Two Options:**

### Option A: Manual Assignment

**Best for:**
- Custom problem selection
- Specific skill practice
- Reusing problems you've used before

**Steps:**
1. Navigate to "Assignments" → "Create Assignment"
2. Select "Manual Selection"
3. Fill in:
   - Title: e.g., "Array Traversal Practice"
   - Description: e.g., "Practice finding max, min, and average in arrays"
   - Class: Select your beta trial class
   - Due date: 1-2 weeks from now
4. Select questions from question bank (3-5 recommended for first assignment)
5. Click "Create Assignment"

**Tips:**
- Start with 1-3 questions for first assignment
- Choose array traversal problems (best supported)
- Avoid complex multi-step problems for beta

### Option B: Auto-Generate Assignment (Recommended)

**Best for:**
- Quick setup
- Balanced difficulty
- Trying multiple skills

**Steps:**
1. Navigate to "Assignments" → "Create Assignment"
2. Select "Auto-Generate"
3. Fill in:
   - Title: e.g., "Mixed Array Skills"
   - Description: e.g., "Auto-generated array practice"
   - Class: Select your beta trial class
   - Skills: Check "Arrays" and/or "Loops"
   - Difficulty: Select "Easy" or "Medium"
   - Question count: 2-3 for first assignment
   - Due date: 1-2 weeks from now
4. Click "Generate Assignment"

**What Happens:**
- System selects questions matching your criteria
- Questions are from your question bank
- Preview shows selected questions
- You can regenerate if not satisfied

**Tips for First Assignment:**
- Start with 2-3 questions
- Choose "Easy" or "Medium" difficulty
- Select "Arrays" skill
- Give students 1-2 weeks to complete

## Step 8: Student Runs Public Tests

**What Students Will Do:**

1. Login with their credentials
2. Navigate to dashboard → Open assignment
3. Click on a question
4. Read the problem prompt
5. Edit the starter code
6. Click "Run Public Tests"

**What Happens During Public Test Run:**
- Student's code is compiled
- Public test cases execute (usually 2-3 tests)
- Results displayed:
  - ✅ "Passed" for correct output
  - ❌ "Failed" with input shown (no expected output)
  - 🔴 "Compilation Error" or "Runtime Error" if code doesn't compile/run
- Partial score calculated (e.g., "4/10 points")

**Why Public Tests?**
- Students can iterate and improve their code
- Learn from test inputs without seeing expected outputs
- Practice debugging and testing
- Reduce anxiety about final submission

**Common Student Questions:**

**Q: "Can I run tests multiple times?"**
A: Yes! Run as many times as needed before final submission.

**Q: "Why don't I see expected output?"**
A: That's intentional. Try to figure out the correct output based on the input.

**Q: "What if I get 0/10?"**
A: Check for compilation errors first, then logic errors. Run tests again after fixing.

**Q: "How many public tests are there?"**
A: Usually 2-3 public tests. Final submission includes hidden tests too.

---

## Step 9: Student Submits Final Answer

**When Student is Ready:**

1. Click "Submit Final Answer"
2. Confirm submission (can't undo)
3. Code is graded against ALL tests (public + hidden)
4. Final score displayed (e.g., "10/10 points")

**What Happens:**
- Code runs against all test cases (typically 5-10 total)
- Hidden tests cover edge cases students didn't see
- Score calculated based on passed tests
- Submission timestamp recorded
- **Cannot resubmit** (final answer is final)

**Why Final Submission?**
- Simulates AP exam conditions (one submission)
- Encourages students to test thoroughly with public tests
- Provides closure and definitive score

**Common Student Questions:**

**Q: "Can I change my answer after submitting?"**
A: No, final submission is final. That's why we have public tests first.

**Q: "What if I submitted too early?"**
A: Unfortunately, you can't resubmit during beta. Use public tests to check thoroughly next time.

**Q: "I got 8/10, what did I miss?"**
A: Hidden tests likely covered edge cases. Check for null arrays, empty arrays, negative numbers, etc.

## Step 10: Teacher Reviews Dashboard Analytics

**Navigate to:** Dashboard page

**What You'll See:**

### 1. Assignment Completion Rates

**Table shows:**
- Assignment title
- Total students in class
- Students who **attempted** (ran any test or submitted)
- Students who **completed** (submitted final answer)
- Attempt rate (% who tried)
- Completion rate (% who finished)
- List of students who didn't complete

**How to Use This:**
- Identify students who haven't started → Send reminders
- Check completion rate → Adjust due dates if needed
- Review "not completed" list → Provide individual support

### 2. Question Performance

**Table shows:**
- Question title
- Submission count
- Average score (across all students)
- Pass rate (% who got full marks)

**How to Use This:**
- Low pass rate → Question may be too hard or unclear
- High variance → Some students need more support
- Zero submissions → Assignment may not be visible

### 3. Performance by Skill

**Table shows:**
- Skill name (e.g., "Arrays", "Loops")
- Average score across all questions with that skill
- Number of questions tagged with that skill

**How to Use This:**
- Identify weak skills → Plan additional practice
- Compare skills → Balance future assignments
- Track improvement over time

## Tips for a Successful Beta Trial

### Start Small
- Begin with 5-10 students, not your entire class
- Create one simple assignment first
- Expand after initial success

### Use Array Problems
- Platform best supports array traversal
- Start with `public int findMax(int[] arr)` style problems
- Avoid complex multi-class problems

### Set Clear Expectations
- Tell students this is a beta trial
- Some bugs are expected
- Feedback is valuable

### Monitor Closely
- Check dashboard daily during first assignment
- Respond to student questions quickly
- Document any issues you encounter

### Communicate Proactively
- Send reminders before due dates
- Share tips based on analytics (e.g., "Watch out for empty arrays")
- Celebrate successes (e.g., "10 students completed!")

### Protect Privacy
- Use anonymized accounts as designed
- Don't share student data in bug reports
- Blur screenshots before sharing

### Be Patient
- Beta software has rough edges
- Report bugs, don't get frustrated
- Your feedback directly improves the tool

## Troubleshooting

### Issue: I Forgot to Save Passwords

**Solution:**
- Bulk-create new accounts (they'll be Student #11, #12, etc.)
- Assign new credentials to students who lost theirs
- This is a known limitation we're fixing post-beta

### Issue: Student Can't Login

**Check:**
- Email typed correctly? (common typo: .demo vs .com)
- Password copied with no extra spaces?
- Caps lock off?
- Try in incognito/private browser window

**If Still Fails:**
- Create a new account for that student
- Report the issue in feedback form

### Issue: Assignment Not Visible to Students

**Check:**
- Assignment created for correct class?
- Assignment published (not draft)?
- Student logged into correct account?
- Refresh browser page

### Issue: Auto-Grading Seems Wrong

**Document:**
- Question title and ID
- Student code submitted
- Expected score vs. actual score
- Screenshot of test results

**Report:**
- Include in feedback form
- Priority issue for beta trial

### Issue: Public Tests Never Finish

**Possible Causes:**
- Infinite loop in student code
- Very slow algorithm (e.g., bubble sort on large array)
- Platform issue (rare)

**Solution:**
- Java runner has 5-second timeout
- Student should see "Timeout" error
- If not, report as bug

### Issue: Platform is Slow

**Check:**
- Internet connection stable?
- Browser up to date?
- Multiple tabs open?

**Report:**
- Time of day (may be server load)
- Specific page (dashboard vs. question)
- Browser type and version
