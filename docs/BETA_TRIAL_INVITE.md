# AP CS Practice Lab Beta Trial Invitation

## What is AP CS Practice Lab?

AP CS Practice Lab is a **beta AP CSA Java FRQ practice platform** designed to help students practice Free Response Questions with immediate auto-grading feedback.

**Core Features:**

- ✅ **Java FRQ Auto-Grading** - Instant feedback on student code submissions
- ✅ **Public & Hidden Tests** - Students can test their code before final submission
- ✅ **Teacher Question Bank** - Create and manage custom Java FRQ problems
- ✅ **Auto-Generated Assignments** - Quickly generate assignments by skill and difficulty
- ✅ **Teacher Analytics** - Track completion rates and question performance
- ✅ **CSV Export** - Export data for record-keeping and grading
- ✅ **Anonymized Student Accounts** - Bulk-create privacy-safe demo accounts

**Technology Stack:**
- FastAPI (Python backend)
- Next.js (React frontend)
- PostgreSQL database
- Docker-based Java code execution

## Current Beta Scope

**What This Platform Supports (Beta v0.1):**

✅ **AP CSA Only**
- Focus on AP Computer Science A curriculum
- Java programming language
- FRQ-style problems

✅ **Array FRQ Practice**
- Array traversal problems
- Array manipulation exercises
- `public int solve(int[] nums)` style methods

✅ **Basic Auto-Grading**
- Public test cases (visible to students)
- Hidden test cases (only for final grading)
- Compilation error feedback
- Runtime error detection

**What This Platform Does NOT Yet Support:**

❌ **AP CSP** - AP Computer Science Principles not yet implemented
❌ **AI Feedback** - No natural language feedback on code quality
❌ **Rubric Scoring** - No AP-style rubric grading (only test-based scoring)
❌ **Advanced Java Features** - No complex OOP, no multi-class problems yet
❌ **Real-Time Collaboration** - Students work independently
❌ **OAuth/SSO** - No Google/Microsoft login integration

**Known Limitations:**
- Desktop browser recommended (not mobile-optimized)
- Docker socket access required for Java runner
- Single-school deployment only
- No password reset mechanism during beta

## What Beta Teachers Will Do

**Expected Activities (2-4 weeks):**

### Week 1: Setup & Familiarization
1. **Login and explore** the platform with demo credentials
2. **Read beta notice** and privacy guidelines
3. **Create a class** for your beta trial group
4. **Bulk-create anonymized student accounts** (5-20 students recommended)
5. **Save temporary passwords** immediately (shown only once)

### Week 2: First Assignment
1. **Create or auto-generate** a Java array assignment (1-3 questions)
2. **Share student login credentials** with your beta group
3. **Instruct students** to:
   - Login with provided credentials
   - Open the assignment
   - Run public tests to check their code
   - Submit final answer when ready
4. **Monitor student progress** on teacher dashboard

### Week 3-4: Analytics & Iteration
1. **Review dashboard analytics**:
   - Which students completed the assignment?
   - Which questions had low pass rates?
   - Which skills need more practice?
2. **Export CSV** for record-keeping if needed
3. **Create a second assignment** (optional)
4. **Gather student feedback** informally

### End of Trial
1. **Complete the beta feedback form** (provided separately)
2. **Share screenshots or screen recordings** (with privacy guidelines)
3. **Suggest improvements** for future releases

## Privacy Expectations

**Protecting Student Privacy During Beta:**

### Use Anonymized Accounts
- ✅ **Do:** Bulk-create accounts with demo emails (`student-001@class-X.demo`)
- ✅ **Do:** Use generic student names (`Student #1`, `Student #2`)
- ❌ **Don't:** Use real student emails
- ❌ **Don't:** Use real student names

### Data Collection
**What This Platform Collects:**
- Login timestamps
- Assignment submissions and scores
- Code submissions (for grading)
- Teacher-created questions and assignments

**What This Platform Does NOT Collect:**
- Student home addresses
- Student phone numbers
- Student photos
- Student payment information
- Any data from external platforms (Google, etc.)

### Screenshots & Demos
- ✅ **Do:** Use anonymized accounts for screenshots
- ✅ **Do:** Blur backend URLs if needed
- ❌ **Don't:** Show real student names in screenshots
- ❌ **Don't:** Show student emails in recordings
- ❌ **Don't:** Show SECRET_KEY, database URL, or tokens

### Classroom Communication
- ✅ **Do:** Share temporary passwords through secure channels
- ✅ **Do:** Explain to students this is a beta trial
- ❌ **Don't:** Post student credentials on public boards
- ❌ **Don't:** Share credentials via unencrypted email if avoidable

### End of Beta
- Platform owner will provide instructions for:
  - Data export (if needed)
  - Account cleanup (if requested)
  - Data retention policy

## Frequently Asked Questions

**Q: Is this free during beta?**
A: Yes, beta trial is completely free. No credit card required.

**Q: Do I need to be a CS expert to participate?**
A: No, if you teach AP CSA or Java, you're qualified. The platform is designed for teachers like you.

**Q: What if I find a bug?**
A: Please report it! Beta trials are for finding bugs. We'll work with you to fix issues quickly.

**Q: Can I use this with my entire class?**
A: We recommend starting with a pilot group (5-20 students) for the beta. You can expand later.

**Q: What if my students already have Google Classroom accounts?**
A: This platform uses its own accounts (anonymized during beta). No Google integration yet.

**Q: Can students reset their passwords?**
A: Not during beta. If a student forgets their password, you can bulk-create a new account.

**Q: Will this replace my AP CSA textbook?**
A: No, this is a **practice and auto-grading tool**, not a curriculum. Use it alongside your existing materials.

**Q: Do I need to install anything?**
A: No installation required. Access via web browser. Backend deployment is handled by the platform owner.

**Q: What if I want to stop using it mid-trial?**
A: No problem. Just let us know. Your feedback will still be valuable.

**Q: Can I suggest new features?**
A: Absolutely! Feature requests are a key part of the beta trial feedback.
