# MCQ Exam Mode Dry Run

Manual checklist for Milestone 4.8.1 hardening and validation.

## A. Teacher Setup

1. Log in as teacher.
2. Import sample MCQ JSON.
3. Confirm questions appear in Question Bank.
4. Create a new assignment with 3-5 MCQ questions only.
5. Assign it to a test class.

## B. Student Run

1. Log in as a student from that class.
2. Open My Assignments.
3. Confirm assignment does not show fake 0/0 progress.
4. Click Start Practice.
5. Confirm MCQ exam mode opens.
6. Answer at least two questions.
7. Mark one question for review.
8. Leave one question unanswered.
9. Submit.
10. Confirm unanswered warning appears before final submission.
11. Confirm score summary appears.
12. Confirm selected answers and correct/incorrect/unanswered statuses appear.

## C. Teacher Review

1. Log back in as teacher.
2. Open Submissions.
3. Filter by assignment.
4. Confirm MCQ submissions appear.
5. Confirm scores are recorded.

## D. Safety Checks

1. Before submission, student cannot see correct answers.
2. Student cannot see `is_correct`.
3. Student cannot see hidden expected outputs.
4. Student cannot see teacher audit notes.
5. Teacher navigation is still available only to teachers.
